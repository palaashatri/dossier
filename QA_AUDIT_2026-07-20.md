## Critical Bug: Scan → Report Navigation Failure

### Bug Description
After a scan completes (`isScanning` transitions from true → false), the expected navigation to Report screen fails. Instead, the app shows the Identity screen.

### Code Evidence
The navigation should work correctly (DossierNavGraph, line 164-182 in MainHubScreen.kt):
```kotlin
composable("scan") {
    ScanScreen(onScanComplete = {
        navController.navigate("report")
    })
}
composable("report") {
    ReportScreen(...)
}
```

ScanScreen correctly signals completion (line 94-101 of ScanScreen.kt):
```kotlin
LaunchedEffect(isScanning) {
    if (isScanning) {
        hasStarted = true
    } else if (hasStarted) {
        liveLogs.add("Scan complete.")
        delay(500)
        onScanComplete()  // Should call navController.navigate("report")
    }
}
```

### Root Cause (TBD — Requires Device Testing)
Possible causes:
1. `onScanComplete()` callback not being called when scan finishes
2. `navController.navigate("report")` silently failing
3. Navigation succeeds but back stack is immediately popped by unknown code
4. NavHost being recreated and resetting to startDestination="identity"

### Diagnostic Additions
Added logging to isolate the failure point:
- **ScanScreen.kt:100** — Logs navigation attempt and state
- **MainHubScreen.kt:164-175** — Logs navigation success/failure and back stack state

These logs will output to logcat and identify exactly where the navigation fails.
